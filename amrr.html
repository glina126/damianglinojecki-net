Introduction

Automatization is beautiful. Seeing your creation come to life is even more beautiful. This project is exactly about that, automatization of the model railroad.

This is just the beginning of a long yet beautiful journey. This project has many goals ahead, but once the goals will be reached, it will shine like a bright star on a dark sky.

Here are some of the goals:

  • Computer map - display a map of the railroad with trains mapped as they move along
  • Automatization - schedule trains and let the computer take care of running them while avoiding collisions
  • Auto train scheduling - let the computer take care of everything, including scheduling, routing and runnning trains
  • Human interaction - let the user pick a train to manually control and make the computer adjust all other traffic accordingly

Given the goals, the project has a very long future. This is a project that will easily take a year to complete. The initial goal might be complete within about 2 months, but it is still very big. There is a lot that goes into the first goal and it has to foresee the future.

Hardware Overview

The hardware is complicated. The PCB layout is easy but the interconnection network is very complex. Even on a simple layout there will be a few microcontrollers.

The Node

A node in the system has 2 uses. It controlls the servos which are used to swich tracks and it serves as a block detector. This node is connected to a master microcontroller via I2C and the master is connceted to a PC via serial communication. The master reads all the nodes in the system as fast as it can and informs the PC of the current status of the blocks. The node has a maximum of two detection blocks connected to it and it reads them all the time. When a change is detected it is stored in the RAM. Once the master requests the information, it is sent over to it. The node is one of the more complicated structures of the automatization system.

The Master

Master serves a purpose of communcating with the PC via serial communication. It tells the PC everything about the system as fast as it possibly can. (most likely at 115200bps) This enables the PC to know everything about every node, detection block and servo in the system and allows for heavy computation to be left to a bigger CPU. The PC serves a role of a controller, i.e the brains of the system. Routing trains and keeping track of everything will require a lot of memory and CPU, thus a microcontroller is not ideal to serve this purpose.

Detection Block

This is the simplest hardware of it all. It is based on a photoresistor and a voltage divider circuit. It is read by node and it detects if a train is over the block or not.
